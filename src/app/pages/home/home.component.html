Role & Permission Rules

Every action on this platform is controlled by the role assigned to your account.

Student (USER) @if (role.includes('USER')) { You }

  • Cannot create, update or delete any student
  • Can view all teachers
  • Cannot create, update or delete any teacher
Teacher (TEACHER) @if (role.includes('TEACHER')) { You }
  • Can view all students
  • Can create new students
  • Can update only the students they created
  • Can delete only the students they created
  • Can view all teachers

Admin (ADMIN) @if (role.includes('ADMIN')) { You }
  • Full CRUD on all students
  • Full CRUD on all teachers
  • Can manage every record on the platform

About this Project

Authify is a full-stack role-based access control (RBAC) system with a secure backend and a modern Angular frontend.

Frontend

Built with Angular 21 (Standalone Components)

  • Angular 21 standalone components with lazy loading
  • Role-based route guards (authGuard)
  • HTTP interceptor for automatic JWT token injection
  • Reactive forms with inline field-level backend error display
  • ChangeDetectorRef for instant UI updates on async data
  • Toast notifications for all user feedback
  • Permission checks on every Create / Update / Delete action
  • Responsive design with vanilla CSS design tokens

Backend

Built with Spring Boot 3 + Spring Security

  • JWT-based stateless authentication (access tokens)
  • Role-based access control: ADMIN, TEACHER, USER
  • Method-level and URL-level permission enforcement
  • Email OTP verification on registration
  • Forgot password flow with OTP via email
  • Password reset with token expiry validation
  • Audit fields on records: createdBy, updatedBy, timestamps
  • Clean RESTful API with structured error responses
  • Deployed on Render (free tier — cold start applies)
